# API Endpoints Documentation 🚀 (Backend v1.0)

This document outlines the current endpoints for the **PasteleriaApi**. The backend implementation is the final authority on available features.

## Base URL
`http://localhost:5000/api` (Local Development)

---

## 🍰 Standard Response Format
All endpoints return a standardized `ApiResponse<T>` object:
```json
{
  "data": T,
  "message": "Success/Error message in English",
  "errors": ["List of errors if any"],
  "isSuccess": true/false,
  "statusCode": 200
}
```

---

## 🥖 Recipe Module (`/Recipe`)
- `GET /Recipe`: Retrieves a list of all recipes (Lightweight, no ingredients included).
- `GET /Recipe/{id}`: Retrieves detailed recipe info, including the list of ingredients.
- `POST /Recipe`: Creates a new recipe. (IDs are generated by the server).
- `PUT /Recipe/{id}`: Updates an existing recipe.
- `DELETE /Recipe/{id}`: Removes a recipe.

---

## 📦 Inventory Item Module (`/InventoryItem`)
- `GET /InventoryItem`: List all stock items.
- `GET /InventoryItem/{id}`: Detail of a specific stock item including Ingredient info.
- `POST /InventoryItem`: Add a new stock item.
- `PUT /InventoryItem/{id}`: Update stock levels or location.
- `DELETE /InventoryItem/{id}`: Remove item from inventory.

---

## 🥕 Ingredient Module (`/Ingredient`)
- `GET /Ingredient`: List all base ingredients.
- `GET /Ingredient/{id}`: Detail of an ingredient.
- `POST /Ingredient`: Create a new base ingredient.
- `PUT /Ingredient/{id}`: Update ingredient info.
- `DELETE /Ingredient/{id}`: Delete an ingredient.

---

## 👤 Auth & Role Module
- `POST /Authentication/Register`: New user registration (Default role: `User`).
- `POST /Authentication/Login`: Authentication returns JWT Token.
- `POST /Authentication/ConfirmEmail`: Confirms email via code.
- `POST /Authentication/ForgotPassword`: Initiates password recovery.
- `POST /Authentication/ResetPassword`: Sets new password via token.
- `GET /Role`: List all Identity Roles (`Admin`, `User`, `Visitor`).

---

## 📄 Document Module (`/Document`)
- `GET /Document`: List all system documents.
- `POST /Document`: Upload/Register a new document.
- `DELETE /Document/{id}`: Remove a document.

---

## 💡 Technical Notes
1. **Guids**: All IDs are `Guid` format. Server-side generation for new records.
2. **Roles**: Seeded roles are `Admin`, `User`, and `Visitor`.
3. **Database**: Currently using **SQLite** for development (`pasteleriapp.db`).
4. **Framework**: Developed with **.NET 8.0**.

  data: T | null;
  message: string;
  errors: string[] | null;
  isSuccess: boolean;
  statusCode: number;
}

export interface ListRecipeDto {
  id: string;
  name: string;
  description: string;

  suggestedPrice: number;

  imageUrl: string;

export interface CreateRecipeDto {
  name: string;
  description: string;
  instructions: string;
  suggestedPrice: number;
  imageUrl: string;
